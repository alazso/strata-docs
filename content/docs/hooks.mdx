---
title: "Hooks"
description: "Permissions, economy, regions, items, and holograms behind one degrade-gracefully registry."
---

The hook layer lets your plugin use third-party integrations **without depending on them**. You call
a Strata interface; Strata owns the adapter, detects availability, and degrades gracefully when the
backing plugin is absent. Direct internal calls are exactly what breaks when an integration changes,
so the abstraction is the fix.

## Resolving a hook

`StrataApi.hooks()` is a `HookRegistry`. `get(type)` returns the best **available** provider for a
capability, or `null` when none are present, so callers degrade instead of crashing.

```java
EconomyHook eco = StrataApi.hooks().get(EconomyHook.class);
if (eco == null) return;                 // no economy installed, degrade
if (eco.has(player, price)) eco.withdraw(player, price);
```

| Capability | Interface | Resolves to |
| --- | --- | --- |
| Permissions / groups | `PermissionHook` | Bukkit-native, LuckPerms |
| Economy | `EconomyHook` | Vault (your own plugin can self-register) |
| Regions | `RegionHook` | WorldGuard, claim plugins |
| Custom items | `ItemHook` | ItemsAdder, Oraxen, Nexo |
| Holograms | `HologramHook` | DecentHolograms, FancyHolograms, CMI |

The permission hook always answers `has(player, node)` (Bukkit-native), and richer providers add
groups, prefix, and meta on top.

## Granting groups and permissions

`PermissionHook` also exposes write operations for reward-style use cases (a voucher that grants a
rank, a temporary perk, or a single permission). The Bukkit baseline cannot persist permission data,
so it returns `false`; LuckPerms performs and saves the change. These hit the permission store, so
run them off the main thread.

```java
PermissionHook perms = StrataApi.hooks().get(PermissionHook.class);
StrataApi.scheduler(plugin).async(() -> {
    perms.addGroup(player, "vip");                          // permanent rank
    perms.addTempGroup(player, "vip", Duration.ofDays(7));  // expiring rank
    perms.setPermission(player, "myplugin.fly", true);      // single permission
    perms.removeGroup(player, "trial");
    perms.unsetPermission(player, "myplugin.fly");
});
```

Each method returns `true` only when the data actually changed and was saved, and `false` (never an
exception) when the provider cannot perform the write or the backing plugin is absent.

## Custom items across providers

Custom items are provider-specific: an ItemsAdder item is not an Oraxen item. So unlike the other
capabilities, you resolve an item against **every** available `ItemHook` rather than a single best
one. Use `all(ItemHook.class)` and take the first provider that recognizes the item or id.

```java
ItemStack icon = null;
for (ItemHook items : StrataApi.hooks().all(ItemHook.class)) {
    if (!items.isAvailable()) continue;
    ItemStack made = items.createItem("ruby");   // provider's own id form
    if (made != null) { icon = made; break; }
}
```

Strata ships providers for ItemsAdder, Oraxen, and Nexo today. Adding another (MMOItems, EcoItems,
ExecutableItems, MythicCrucible, HeadDatabase) is a new `ItemHook` registration, and the loop above
keeps working unchanged. A provider whose plugin is absent reports `isAvailable()` as `false` and is
skipped.

## Registering your own provider

If you write a plugin that *provides* a capability (say, your own economy plugin), don't make Strata
depend on it. Have it **self-register** on enable:

```kotlin
// In your plugin's onEnable, after Strata is enabled:
StrataApi.hooks().register(EconomyHook::class.java, MyEconomyHook(), priority = 100)
```

Strata stays agnostic, and the registry resolves between providers automatically.

## Preference (non-interchangeable providers)

Economy backends are *different money stores*, so when two are present the admin must choose which is
authoritative, because Strata doesn't guess. `setPreference` honors a configured provider name
(matched case-insensitively against `Hook.name()`) when available, falling back to priority
otherwise.

```java
StrataApi.hooks().setPreference(EconomyHook.class, "vault");
```

Strata applies this from its own `config.yml` (`economy.provider`) on enable.

<Callout type="info">
  Adapters hold **no** third-party fields, only a presence flag, and they wrap all third-party calls.
  So a hook loads and registers fine even when its plugin isn't installed, and reports
  `isAvailable()` as `false` instead of throwing.
</Callout>
